# Instructions pour les agents de développement

- Respecter `docs/charte.md`.
- La trace écrite est la référence pédagogique.
- Ne jamais modifier un exercice déjà validé sans proposition préalable.
- Utiliser la dernière version stable de GeoLib indiquée dans `docs/avancement.md`.
- Les exercices sont des fichiers HTML autonomes, fond transparent, responsive.
- Le titre n'apparaît jamais dans le HTML.
- Toute fonctionnalité destinée à plusieurs exercices doit être développée dans GeoLib.
- Préférer un code simple, robuste et maintenable.
