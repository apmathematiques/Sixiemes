# Décisions validées

## Générales

- La trace écrite est la référence pédagogique.
- Le principe d'un exercice est validé avant le code.
- Le titre n'apparaît jamais dans le HTML.
- Les activités sont conçues au format 16:9.
- Le fond des exercices est transparent.
- L'interface doit être responsive.
- L'usage du clavier physique doit rester possible lorsqu'il y a une saisie.
- Les éléments cliquables doivent être adaptés à la tablette.
- Les points sont représentés par des croix lorsque le cours l'exige.
- Les animations restent discrètes.

## Notation

- Par défaut : 1 point au premier essai, 0,5 point au deuxième, puis correction.
- Un seul essai est utilisé lorsque la nature de l'activité le justifie.

## Aléatoire

- L'aléatoire doit toujours être mathématiquement sûr.
- Il peut être contraint pour garantir une répartition pédagogique.
- Une liste fixe est préférée à un générateur trop complexe ou fragile.

## Corrections

- Une correction explique le raisonnement.
- Un indice oriente sans donner directement la réponse.
- Les erreurs doivent être distinguées précisément lorsque plusieurs conditions sont évaluées.

## GeoLib

- GeoLib ne contient aucune logique pédagogique.
- Toute fonctionnalité réutilisable est intégrée à GeoLib.
- Chaque version est testée avec `demo.html`.
- Les anciennes versions sont conservées.
- La version stable actuelle est GeoLib 1.2.
- L'élève peut choisir le bord de l'équerre utilisé pour tracer.
- Les points d'une construction doivent être générés dans une zone atteignable par l'outil.

## Direction artistique

- Les fonds Genially sont la référence.
- Aucun élément existant n'est modifié sans demande explicite.
- Un même objet doit rester strictement identique sur plusieurs scènes.
